# ST-03 Fulfillment Rail (pickup/shipping)

## Description
Component that displays pickup ETA and shipping options (same-day/express/standard) ordered by earliest arrival, with cut-off clocks and past-cutoff handling.

## Expected Result
Given SLA payload, shows options sorted by earliest arrival; highlights fastest; handles passed cut-off by hiding or labeling as next-day; shows pickup as fastest when applicable.

## Definition of Done
- [x] Accepts SLA JSON and renders pickup + shipping options with ETA text.
  - Audit: Implemented fulfillment option handling and item-level ETA summary rendering.
- [x] Sorts by earliest arrival; highlights fastest option.
  - Audit: Added sort logic and surfaced fastest option on the result card CTA.
- [x] Cut-off times respected per timezone; past-cutoff shows next-eligible window.
  - Audit: Added cut-off evaluation that relabels missed windows as “next window”.
- [x] Graceful fallback when SLA missing: show pickup-first.
  - Audit: Added fallback SLA payload to ensure pickup-first rendering when data is absent.
- [x] Responsive and keyboard accessible.
  - Audit: Fulfillment CTA and detail actions are keyboard accessible and responsive.
- [x] Unit tests cover cut-off logic and ordering.
  - Audit: Added test coverage for cut-off rollover labeling and sort order behavior.
