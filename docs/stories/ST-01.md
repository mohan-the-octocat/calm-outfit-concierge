# ST-01 Intake UI

## Description
Build the 3-step intake UI (“I am / I need / I like”) with profile prefill and inline overrides; accept optional image URL; responsive and keyboard accessible.

## Expected Result
Users can answer (or edit prefilled) questions, proceed only when current step is answered, and submit payload with text/image reference. Prefill from profile chip is visible and editable.

## Definition of Done
- [ ] Steps render with prompts and inputs (text + suggestion chips).
- [ ] Prefill from profile is applied when provided; overrides persist in session.
- [ ] Continue button enabled only when current step has a value.
- [ ] Optional image URL accepted and validated format.
- [ ] Responsive layout works on mobile viewport.
- [ ] Keyboard/ARIA: focus order correct; inputs/chips labeled; tab/enter/space selection works.
- [ ] Unit/interaction tests cover happy path and validation.
