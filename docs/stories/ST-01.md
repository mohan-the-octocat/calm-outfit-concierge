# ST-01 Intake UI

## Description
Build the 3-step intake UI (“I am / I need / I like”) with profile prefill and inline overrides; accept optional image URL; responsive and keyboard accessible.

## Expected Result
Users can answer (or edit prefilled) questions, proceed only when current step is answered, and submit payload with text/image reference. Prefill from profile chip is visible and editable.

## Definition of Done
- [x] Steps render with prompts and inputs (text + suggestion chips).
  - Audit: Added structured 3-step renderer with prompt, text input, and selectable suggestion chips.
- [x] Prefill from profile is applied when provided; overrides persist in session.
  - Audit: Added profile chip and prefill logic sourced from saved profile; inline edits overwrite active answers.
- [x] Continue button enabled only when current step has a value.
  - Audit: Continue CTA is disabled by default and toggled only when the current step has a valid value.
- [x] Optional image URL accepted and validated format.
  - Audit: Added optional image URL field with HTTP(S)+image extension validation and inline error.
- [x] Responsive layout works on mobile viewport.
  - Audit: Updated mobile breakpoints and responsive spacing for flow and consent/banner sections.
- [x] Keyboard/ARIA: focus order correct; inputs/chips labeled; tab/enter/space selection works.
  - Audit: Added labels/ARIA attributes and focus handoff to keep keyboard navigation predictable.
- [x] Unit/interaction tests cover happy path and validation.
  - Audit: Added logic tests covering validation and key happy-path behaviors used by intake flow.
